## アプリの目的

本アプリは、少年野球の試合を初心者でも入力しやすくするスコア入力アプリである。

主な対象ユーザーは、野球やスコアブックに詳しくない保護者を想定する。

現在の中心思想：

- ユーザーに早稲田式の記号を直接入力させない
- 投球ボタン、フィールド上の守備位置、ランナー操作から試合状況を入力する
- 入力結果から、紙のスコアブックに近いスコアマスを自動描画する
- 試合中に迷わない操作を優先しつつ、表示はできるだけ本物のスコアマスに寄せる

## 現在の画面構成

### オーダー

現在実装されている内容：

- 自チームと相手チームのオーダーを表示する
- 自チームは初期データとして東村山ドリームの選手が入っている
- 相手チームは名前、守備位置、打席、背番号を入力できる
- 行のドラッグで打順を並べ替えられる
- 現在の打者行をハイライトする
- 守備位置番号の重複を警告する
- 自チームの先攻 / 後攻を選べる
- 試合開始後は自チームの先攻 / 後攻を変更できない

制約：

- 自チーム選手の名前と背番号は、現状は固定データとして扱う
- 自チームの左右打席は固定表示で、オーダー画面からは編集しない
- 相手チームは最大21行分の入力枠を持つ

表示されるもの：

投球ボタン：

- ストライク

- デッドボール

プレー入力後は、原則として「確定」または「取り消し」を選ぶ。

### スコア出力

現状は「スコア出力」と表示するプレースホルダーである。

## データと状態管理

現在の状態は React のメモリ上の `AppState` で管理している。

主な状態：

- 自チーム、相手チーム
- 自チームオーダー、相手チームオーダー
- イニング
- 表 / 裏
- ボール、ストライク、アウト
- 自チーム得点、相手チーム得点
- 自チームと相手チームそれぞれの現在打順
- 現在の打者・投手の背番号
- 一塁、二塁、三塁のランナー
- 現在打席の投球履歴と結果
- 現在打席のスコアマス情報

永続保存はまだ実装していない。

PWA 用の Service Worker 登録はあるが、試合データの保存機能ではない。

## 選手管理

### 自チーム

現在の自チームは、`src/data.ts` の初期データを使う。

実装済み：

- 先発9人と控え選手を保持する
- 自チーム打撃時にバッター変更ダイアログを開ける
- 控え選手を選ぶと、現在打順の位置に入れ替える
- 自チーム守備時にピッチャー交代ダイアログを開ける
- ピッチャーを選ぶと、その選手の守備位置を `1` にし、他の `1` を外す

### 相手チーム

実装済み：

- オーダー画面で相手チーム名を入力できる
- 相手選手の名前、守備位置、左右打席、背番号を入力できる
- 相手チーム打撃時、現在打者の背番号が未登録なら登録ダイアログを出す
- 相手チーム投手は、登録済み背番号から選ぶか、新しい背番号を入力できる

現状未実装：

- 背番号不明を `?` として正式に保持する入力
- 選手ごとの打撃成績集計
- 守備交代の履歴管理

## 投球入力

投球記号：

- ストライク：`×`
- ボール：黒丸
- ファール：三角
- デッドボール：投球記号は追加せず、結果を `HP` として扱う

処理：

- ストライクはカウントを増やす
- 3ストライクで `K`、アウトカウント +1、アウト番号を記録する
- ファールは投球履歴に残す
- 2ストライク後のファールではストライクカウントを増やさない
- ボールはカウントを増やす
- 4ボールで打者を一塁へ進め、結果を `B` とする
- デッドボールで打者を一塁へ進め、結果を `HP` とする
- 四球 / 死球で押し出しが発生する場合、走者を進めて得点を加算する

投球またはプレー入力後は、確定前の状態になる。

確定：

- 打席が完了している場合、次打者へ進む
- 3アウトの場合、走者を消し、表裏を切り替える
- 裏から表に切り替わるときはイニングを +1 する
- 打席がまだ完了していない通常の1球入力では、カウントと投球履歴を維持して入力待ちに戻る

取り消し：

- 現在の未確定入力を、入力直前のスナップショットへ戻す
- 確定済みの過去入力を1球ずつ戻す履歴機能ではない

## フィールド入力

フィールド上には以下の入力ターゲットがある。

- 守備位置 `1` から `9`
- 一塁、二塁、三塁、本塁
- 左右ファールゾーン
- レフトオーバー、センターオーバー、ライトオーバー
- 打者走者アイコン
- 塁上ランナーアイコン

### 打球入力

守備位置を最初にタップすると、そこへ飛んだ打球として扱う。

現在の動き：

- 打球線を描画する
- 打球方向の守備者番号を記録する
- 打者走者を一塁へ自動進塁させる
- スコアマスには一塁への進塁線と打球方向番号を描画する
- 打球ノードにはアウト / エラーの判定バブルを出せる

初期打球でアウトを選ぶと、フライアウト系として扱う。

ゴロアウト系は、守備位置からベースへ送る流れを入力し、ベース上のアウト判定として扱う。

### 送球とアウト

守備位置やベースを続けてタップすると、送球・走塁の線を描画する。

ベース上の判定では、主に以下を扱う。

- アウト
- エラー

打者走者が一塁でアウトになった場合、守備者番号と送球先から `6-3`、`3A`、`3-1A` などの表記を生成する。

先行走者のアウトは、現状では走塁死として `T.O` を使う。

### ランナー操作

打者走者または塁上ランナーをドラッグして、進塁先へ落とせる。

通常の走者進塁理由：

- 盗塁
- 捕逸
- ボーク
- 走塁妨害

打者走者に対して選べる進塁理由：

- 振り逃げ
- 捕手妨害

振り逃げは、2ストライク以上かつ一塁が空いている、または2アウトのときに使える。

打球入力後のランナー進塁は、原則として安打による進塁として扱う。

### ファールとファールフライ

ファールゾーンをタップすると、ファールを投球履歴に追加する。

その後に守備位置をタップすると、ファールフライアウトとして扱い、`F5` などの表記を生成する。

### 本塁打

外野オーバーターゲットを最初に選ぶと、本塁打候補として扱う。

ホームラン判定を選ぶと：

- 結果を `本` とする
- `hitType` を `home-run` にする
- 塁上ランナーと打者走者を全員得点させる
- 塁上ランナーを消す
- スコアマスには本塁までの進塁線を描画する

## スコアマス表示

スコアマスは `assets/score_matrix.png` を下敷きにし、SVG で記号や線を重ねる。

表示対象：

- 現在打者のスコアマス
- 塁上ランナー本人のスコアマス

現在の表示内容：

- 投球記号
- 進塁線
- 打球方向番号
- 内野安打の半円
- 打撃結果
- アウト番号 `I` / `II` / `III`
- 打席未完了で3アウトになった位置を示す `//`
- 走者の進塁理由
- 走塁死などのアウト表記

### 表記ルールの基準

スコアマスの書き方は、原則として以下の記事の説明に従う。

- https://baseball-one.com/blog/archives/274598/

ただし、現状の実装は記事の全表記を網羅していない。

現状で対応している範囲は `docs/scorebook-rules/ui-coverage.md` と `docs/scorebook-rules/article-diff-audit.md` も参照する。

### 打撃結果と進塁表記

現在の表記：

- 四球：`B`
- 死球：`HP`
- 捕手妨害：`IF`
- 失策：`E`
- 三振：`K`
- 振り逃げ：`K 2-3`
- ゴロアウト例：`6-3`
- 一塁手が自分で踏んだアウト：`3A`
- 一塁手から投手カバー：`3-1A`
- フライアウト：守備者番号
- ファールフライ：`F` + 守備者番号
- 盗塁：`S`
- 捕逸：`P`
- ボーク：`BK`
- 走塁妨害：`OB`
- 走塁死：`T.O`

### アウト番号とイニング終了斜線

アウト番号 `I` / `II` / `III` は、アウトになった本人のスコアマスに書く。

例：

- 2アウトで一塁ランナーが盗塁し、二塁でアウトになった場合
- その一塁ランナーが出塁した打者のマスに `III` を書く
- そのとき打席中だったバッターのマスには `III` を書かない

打席中のバッターが打席完了しないまま3アウトでチェンジになった場合、そのバッターのマスには右下に `//` を書く。

意味の整理：

- `III`：3つ目のアウトになった人を示す
- `//`：そこで攻撃イニングが切れた位置を示す

現状の実装では、先行走者アウトの詳細表記はまだ `T.O` 寄りで、盗塁死の `2-6` などの送球経路表記は限定的である。

日本語の結果ラベルは、スコアマス上では原則として抑制する。

例：

- `安打` は文字として表示せず、進塁線と打球方向番号で表す
- `振り逃げ` は文字として表示せず、`K 2-3` として表す
- `捕手妨害` は文字として表示せず、`IF` として表す

### 安打

安打は、赤い進塁線と打球方向の守備者番号で表す。

現在の実装：

- 守備位置タップで打球方向番号を保持する
- 打者走者を一塁へ自動進塁させる
- スコアマスの一塁側に打球方向番号を表示する
- 守備者番号 `1` から `6` は内野安打として、数字を半円で囲む
- 守備者番号 `7` から `9` は外野への安打として、半円では囲まない

内野安打の半円：

- 数字と重ならないように描画する
- 数字の右下側から包む半円にする
- 半円は、記事中の内野安打例に近い見た目を目標にする

### 長打

現状では、UIから `two-base` / `three-base` を明示的にセットする処理はない。

ただし、打者走者やランナーを二塁・三塁・本塁へ動かした結果として、複数塁への進塁線は描画できる。

未実装：

- `左2` / `中2` / `右2`
- `左3` / `中3` / `右3`
- 打球が「前」「越え」「線上」などだったことの明示
- 長打種別としての二塁打・三塁打の確定入力

本塁打だけは専用処理があり、外野オーバーからホームラン判定を選べる。

## 実装上の注意

今後の変更では、以下を守る。

- 入力UIでは、できるだけ初心者向けの言葉と操作を使う
- スコアマス表示では、紙スコアに近い記号・位置関係を優先する
- 新しい記録種別を追加するときは、`scoreRules.ts` の状態更新と `ScoreMatrixGraphic` の描画をセットで考える
- 記事の表記に寄せる場合も、現在未実装の表記を実装済みのように文書へ書かない
- `docs/scorebook-rules/ui-coverage.md` と矛盾しないようにする

## 現在のプロダクトの要約

現在のアプリは、フィールド操作で試合中のプレーを入力し、その場で現在打者と塁上ランナーのスコアマスを描画する入力画面である。

完成済みスコアブックや出力機能はまだ中心ではない。

いま一番大事な軸は以下。

> 入力はフィールドとランナー操作で簡単にする。  
> 表示は Baseball One の記事に沿った紙スコア表記へ寄せる。  
> ただし、未実装の記録は未実装として明示する。
